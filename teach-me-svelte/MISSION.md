# Mission: Understanding Svelte (via a Vibecoded App)

## Why
Vojtech has a working Svelte 5 / SvelteKit music-interval-training app that was vibecoded (AI-generated). He wants to understand the code well enough to navigate it confidently and make manual changes without relying on AI to modify it for him.

## Success looks like
- Can look at any `.svelte` file in the app and explain what each section does
- Can make targeted changes (add a button, tweak layout, change behavior) without breaking things
- Understands the file/folder structure well enough to know *where* to make a change
- Can read TypeScript annotations in the codebase without confusion

## Constraints
- No prior Svelte experience -- everything was AI-generated
- Shaky TypeScript understanding -- TS should be explained when encountered, not assumed
- Teaching should be grounded in the actual music-interval-training app code, not abstract examples

## Out of scope
- Building a new Svelte app from scratch (for now)
- Server-side rendering, deployment, advanced SvelteKit features
- Deep TypeScript mastery -- just enough to read what's in the app
