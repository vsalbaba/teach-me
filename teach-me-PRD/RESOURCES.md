# PRD Resources

## Knowledge

- [Article: "How to Write a Good PRD" -- Ulad Shauchenka](https://www.uladshauchenka.com/p/how-to-write-a-good-product-requirements)
  Best single-article overview. Covers all 14 PRD sections, Gherkin acceptance criteria, NFRs via ISO 25010, and common mistakes. Includes a full Markdown template.

- [Article: "The Only PRD Template You Need" -- Product School](https://productschool.com/blog/product-strategy/product-template-requirements-document-prd)
  Concise template with worked example. Good for comparing against the Shauchenka template.

- [Article: "How to Write a PRD" -- Perforce](https://www.perforce.com/blog/alm/how-write-product-requirements-document-prd)
  Broader ALM context. Covers PRD vs MRD, waterfall vs agile PRDs, and traceability.

- [DeepWiki: Breaking PRDs into Vertical Slices -- Matt Pocock's Skills](https://deepwiki.com/mattpocock/skills/3.2-breaking-prds-into-vertical-slices)
  Five-step process for decomposing a PRD into vertically-sliced issues. Covers tracer bullets, dependency ordering, and issue templates.

- [GitHub: Matt Pocock's to-prd skill](https://github.com/mattpocock/skills/blob/main/skills/engineering/to-prd/SKILL.md)
  Claude Code skill that converts design docs into PRDs. Useful as a reference for PRD structure expectations in engineering workflows.

- [Article: "Writing PRDs and Product Requirements" -- Carlin Yuen (Medium)](https://carlinyuen.medium.com/writing-prds-and-product-requirements-2effdb9c6def)
  Practitioner perspective on combining PRDs with user stories. Good for the "when to use what" question.

- [Article: "PRD Templates and Examples" -- AltexSoft](https://www.altexsoft.com/blog/product-requirements-document/)
  Detailed walkthrough of PRD sections with visual examples. Engineering-friendly.

## Wisdom (Communities)

- [r/ProductManagement](https://reddit.com/r/ProductManagement)
  Active subreddit for product managers. Good for PRD critique and real-world examples.

- [Lenny's Newsletter community](https://www.lennysnewsletter.com/)
  High-signal product management content. Paid, but free articles cover PRD practices.

## Gaps

- No open-source repository of real, complete PRDs to study (most are behind paywalls or NDAs)
- Limited resources on PRD practices specifically for developer-PMs or QE-adjacent roles
